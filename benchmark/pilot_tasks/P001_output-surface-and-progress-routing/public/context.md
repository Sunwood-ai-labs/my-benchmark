# 関連ファイルと前提
- イベント発火点、チャット送信関数、CLI ログ関数が repo 内にある。
- README か既存テストに期待される利用体験が書かれている。
- debug 用の verbose ログは完全には不要ではない。

# 最小仕様
- チャット側には人間が読める進捗のみを送る。
- CLI 側には原因調査に必要な詳細を残す。
- 同一イベントが二重送信されない。

# fixture について
- fixture は bot のログイベント列と送信先サーフェスの差分を再現できる最小構成でよい。
- 外部接続は不要。

# 評価上の注意
- 出力先の分離を UI redesign と取り違えないこと。
- ユーザー向けメッセージに raw debug を混ぜると hard fail に近い減点になる。
