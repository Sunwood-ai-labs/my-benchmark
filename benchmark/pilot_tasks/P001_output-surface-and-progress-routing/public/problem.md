# User Prompt
長時間ジョブの出力面を直してください。CLI では進捗と診断を追えるようにし、チャット relay には人間向けの短い進捗だけを出してください。今の debug 混入はそのまま流さないで、何を CLI に残して何をチャットへ出すかが分かる確認も付けてください。

# Constraints
- 主処理フローや message schema を全面改修しないこと。
- ログを黙らせるだけで終わらないこと。
- token、内部識別子、raw payload をチャット向け出力へ混ぜないこと。

# Output
- 変更したファイル
- CLI 面とチャット面で何が見えるかの確認
- 変更理由の短い説明
