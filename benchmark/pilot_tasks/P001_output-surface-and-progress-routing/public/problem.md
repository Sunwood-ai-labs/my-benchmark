# Goal
詳細デバッグは CLI に残しつつ、ユーザー向けチャンネルには短い進捗だけを出すように最小差分で直す。

# Background
現在の実装では低レベルの診断ログがチャット側に流れ、逆に CLI 側では欲しい進捗が見えにくい。

# Input Context
- CLI 出力とチャット送信が同じイベント列から組み立てられている。
- 既存テストは送信の有無を見ているが、出力面の適合までは見ていない。
- ユーザーは「見やすさ」と「進捗把握」を両方要求している。

# User Prompt
既存のログ経路を読み、CLI とチャットで出す情報を整理してください。チャットには短い進捗、CLI には詳細診断を残し、どこをどう分けたかが分かる検証を付けてください。

# Constraints
- bot の主機能やメッセージ本文フォーマットを全面的に作り直さないこと。
- 秘密情報や内部識別子をユーザー向け出力に混ぜないこと。
- 症状隠しのために単純にログを消すだけで終わらないこと。

# Deliverables
- 最小差分の実装修正
- CLI とチャットの想定出力差が分かる検証
- なぜその分離が妥当かの短い説明
