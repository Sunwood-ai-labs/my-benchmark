# Goal
成功判定を「送れた」段階で止めず、履歴や成果物まで確認してから完了扱いにする。

# Background
既存の検証は途中段階で成功としており、ユーザーは履歴や最終状態が本当に残っているかを疑っている。

# Input Context
- 既存の e2e または smoke は prompt 送信か応答検出で終わっている。
- 本当に欲しいのは、履歴・保存物・最終レスポンスなど末端状態の確認。
- ユーザーは「最後まで行っているか」を明示的に要求している。

# User Prompt
既存テストの成功条件を見直し、最後に必要な履歴または成果物が取れていることまで確認してください。何をもって完了とするかも短く示してください。

# Constraints
- 存在しない遠隔依存をでっち上げないこと。
- 再現不能なら blocked と明示し、代替証拠を区別すること。
- 大規模な harness 作り直しは避けること。

# Deliverables
- 成功条件を強化したテストまたは focused check
- 末端状態を示す証拠ポイント
- blocked 時の扱いを含む説明
