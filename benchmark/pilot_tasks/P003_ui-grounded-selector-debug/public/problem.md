# Goal
実際の DOM / UI 状態を観測して、無関係なレイアウトを壊さずにセレクタまわりの不具合を直す。

# Background
抽象的な推測だけでレイアウトやセレクタをいじると、症状の本体を外したまま UI を壊しやすい。

# Input Context
- 既存の selector fallback や diagnostic script がある。
- ユーザーは「ちゃんとレイアウトを認識しろ」と強く要求している。
- 無関係な UI 調整はむしろ回帰リスクになる。

# User Prompt
実際の UI 状態を確認し、どの前提が外れていたかを特定してから、最小差分でセレクタまたは探索ロジックを直してください。レイアウト側に不要な変更は入れないでください。

# Constraints
- スクリーンショット、DOM、診断ログなどの実観測なしで推測修正しないこと。
- 見た目のレイアウト調整で誤魔化さないこと。
- 無関係な CSS / UI 文言変更を混ぜないこと。

# Deliverables
- 実観測に基づく最小差分修正
- どの前提が誤っていたかの説明
- UI を壊していない確認
