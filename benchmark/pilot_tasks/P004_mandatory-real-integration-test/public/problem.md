# User Prompt
ローカル test や CLI 確認だけで verified にしないでください。real-surface equivalent でどこまで確認できるか切り分けて、可能ならそこまで確認してください。必要な外部設定が足りないなら blocked と next step を明示してください。

# Constraints
- 確認していないことを verified と書かないこと。
- 危険な外部操作や本番面への書き込みはしないこと。
- 単体テスト pass や help 出力だけをもって完了扱いしないこと。

# Output
- 実際に確認した面
- blocked の有無と理由
- 既存テストとの関係
