# Goal
単体やローカル smoke だけで済ませず、実チャネルに相当する面で確認するか、できないなら blocked を明確にする。

# Background
ユーザーは単体成功だけでは納得せず、実際の利用面での確認を明示的に求めている。

# Input Context
- ローカル test は通るが、実チャネル相当の確認が別経路にある。
- ユーザーは「必須」と表現しており、単なる nice-to-have ではない。
- ただし外部依存の制約で完全再現できない場合もある。

# User Prompt
どこまでがローカル確認で、どこからが実チャネル確認なのかを切り分けてください。安全に可能な範囲で実面確認を行い、無理な部分は blocked と next step を明示してください。

# Constraints
- 確認していないことを verified と書かないこと。
- 危険な外部操作はしないこと。
- 単体テスト pass をもって完了扱いしないこと。

# Deliverables
- 実チャネル相当の確認か、正直な blocked レポート
- 境界の説明
- 既存テストとの関係整理
