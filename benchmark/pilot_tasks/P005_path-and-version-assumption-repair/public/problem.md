# Goal
デフォルト前提で進めず、実際のパスと現行バージョンを確認してから修正方針を決める。

# Background
誤った配置先や古い互換前提で進めると、実装自体が合っていてもユーザーには外れた対応に見える。

# Input Context
- README、config、ログ、version metadata が局所に存在する。
- ユーザーは「ここじゃないの？」「最新版でも使えるようにした？」と前提修正を求めている。

# User Prompt
実際の配置先と現行バージョンを先に確認し、その前提に合わせて修正か説明を行ってください。デフォルト値や古い例を鵜呑みにしないでください。

# Constraints
- ローカル証拠より一般論を優先しないこと。
- 実際に確認していない path/version を断定しないこと。

# Deliverables
- 修正済みの前提整理
- path/version に合った変更または説明
- 確認に使ったローカル証拠
- 変更したファイル名、または変更不要と判断した理由
- 実行した確認手順
