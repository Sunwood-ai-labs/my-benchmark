# Accepted Fix

報告された不具合を最小限の差分で直し、既存の挙動や周辺構造をむやみに壊さないことを重視する。
- 出力が読める形に戻り、構造も意図通りになる。

# Minimum Pass

- 主症状が解消し、最低1つの妥当な検証が示されていること。

# Acceptable Variants

- 本質的に同じ原因線を直していれば、実装の細部は多少異なってもよい。

# Common Failures

- 無関係なファイルまで巻き込んでいる。
- 検証せずに直ったと断言している。
- 症状の回避だけで根本原因が残っている。

# Evaluator Notes

- 症状の抑え込みではなく、報告内容に対して原因を外していないかを見る。
