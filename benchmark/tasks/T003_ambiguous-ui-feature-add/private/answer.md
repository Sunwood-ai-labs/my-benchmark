# Expected Outcome
不足している仕様を安全に補いながら、使い物になる機能を実装できるかを見る。
- まとまりのある UI 改善が入り、実装と説明が噛み合っている。

# Strong Answer Characteristics
- 質問で止まりすぎず、妥当な前提を置いて前進している。
- 既存のUIやCLIの流れに自然に馴染む。
- 過剰実装に走らず、まとまりのある着地をしている。

# Acceptable Variants
- 複数の着地があり得るが、一貫した前提と妥当な制約が必要。

# Common Failure Patterns
- 細部の不明点で過度に停止する。
- 要求より大きすぎる変更を入れる。
- 既存の構造や体験と噛み合わない実装をする。

# Minimal Pass Line
- 機能が成立し、置いた前提が読めること。

# Notes For Evaluator
- 大きい答えより、まとまりのある答えを高く評価する。
