# Accepted Fix

不足している仕様を安全に補いながら、使い物になる機能を実装できるかを見る。
- まとまりのある UI 改善が入り、実装と説明が噛み合っている。

# Minimum Pass

- 機能が成立し、置いた前提が読めること。

# Acceptable Variants

- 複数の着地があり得るが、一貫した前提と妥当な制約が必要。

# Common Failures

- 細部の不明点で過度に停止する。
- 要求より大きすぎる変更を入れる。
- 既存の構造や体験と噛み合わない実装をする。

# Evaluator Notes

- 大きい答えより、まとまりのある答えを高く評価する。
