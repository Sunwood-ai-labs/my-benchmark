# Goal
仕様が足りない状態で、まとまりのある UI 改善を実装する。

# Background
利用者は『もっとリッチに』『無難すぎない見た目に』とだけ言っており、完全な wireframe はない。

# Input Context
- 小さな Web か app UI があり、主操作は1つに絞られている。
- 改善要求は具体性が足りないが、実装は期待されている。

# User Prompt
過度に質問で止まらず、妥当な前提を置いて UI 改善を実装し、前提は最後に短く明示してください。

# Constraints
- 安全上の問題がない限り、質問待ちで停止しないこと。
- 一貫した小さな改善に留めること。

# Deliverables
- 動く UI 改善。
- 前提の短いメモ。
