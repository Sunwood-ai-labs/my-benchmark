# Accepted Fix

コードではなく実行経路や設定の問題を見極め、既存ツールチェーンで復旧できるかを見る。
- mismatch の原因が塞がれ、次回以降もズレにくくなる。

# Minimum Pass

- 壊れていた実行経路が復旧し、repo 標準の方法で確認できること。

# Acceptable Variants

- 設定修正、起動経路修正、workflow 修正のいずれでも、原因に合っていれば可。

# Common Failures

- 間違ったランタイムやパッケージマネージャを使う。
- 本質原因に触れず、その場しのぎで通す。
- 再現性のない直し方をする。

# Evaluator Notes

- 依存関係の大掃除で押し切っていないかを確認する。
