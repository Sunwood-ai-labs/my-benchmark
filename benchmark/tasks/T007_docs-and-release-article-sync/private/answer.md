# Expected Outcome
ローカルの差分や成果物に基づき、説明面を正確に整えられるかを見る。
- 説明が local change set と整合し、過剰な主張がない。

# Strong Answer Characteristics
- git 差分やローカル成果物に根拠を持っている。
- 関連する docs surface のズレを埋めている。
- 余計な全面書き換えに広げていない。

# Acceptable Variants
- README、release note、記事、図表のどれを直しても、与件に合っていれば可。

# Common Failure Patterns
- 根拠のない説明を書いている。
- 片方だけ直して整合が崩れる。
- 文体整理に脱線する。

# Minimal Pass Line
- 主要なズレが解消し、説明がローカル事実と一致していること。

# Notes For Evaluator
- 文章量より、根拠との一致を優先する。
