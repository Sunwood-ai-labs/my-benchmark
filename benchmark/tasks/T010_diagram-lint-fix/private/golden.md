# Accepted Fix

ローカルの差分や成果物に基づき、説明面を正確に整えられるかを見る。
- 報告されていた lint failure が解消する。

# Minimum Pass

- 主要なズレが解消し、説明がローカル事実と一致していること。

# Acceptable Variants

- README、release note、記事、図表のどれを直しても、与件に合っていれば可。

# Common Failures

- 根拠のない説明を書いている。
- 片方だけ直して整合が崩れる。
- 文体整理に脱線する。

# Evaluator Notes

- 文章量より、根拠との一致を優先する。
