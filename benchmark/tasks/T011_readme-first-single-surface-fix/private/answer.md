# Expected Outcome
報告された不具合を最小限の差分で直し、既存の挙動や周辺構造をむやみに壊さないことを重視する。
- README 由来の制約を踏まえて最小差分で直せる。

# Strong Answer Characteristics
- 症状ではなく原因線に当たり、変更範囲を必要最小限に抑えている。
- 既存のコマンドやテストに沿って検証している。
- 不要な整理や別件の修正を混ぜていない。

# Acceptable Variants
- 本質的に同じ原因線を直していれば、実装の細部は多少異なってもよい。

# Common Failure Patterns
- 無関係なファイルまで巻き込んでいる。
- 検証せずに直ったと断言している。
- 症状の回避だけで根本原因が残っている。

# Minimal Pass Line
- 主症状が解消し、最低1つの妥当な検証が示されていること。

# Notes For Evaluator
- 症状の抑え込みではなく、報告内容に対して原因を外していないかを見る。
