# Goal
README に書いてある前提を読めばすぐ直せる小さな不具合を直す。

# Background
repo は既に正しい運用方法を書いているが、README を読まずに入ると遠回りしやすい。

# Input Context
- README の中に重要な制約が明示されている。

# User Prompt
README を先に読み、その制約を使って局所修正を行ってください。余計な変更はしないでください。

# Constraints
- README を読まずに推測で進めないこと。

# Deliverables
- 小さな修正。
- README のどの手掛かりが効いたか1文。
