# Expected Outcome
ローカル証拠に基づいて状況を説明し、次に取るべき行動を明確にできるかを見る。
- 短くても、見るべき状態と失敗条件が具体的に書けている。

# Strong Answer Characteristics
- 観測事実と仮説を分けている。
- 読むべきファイルや試すべきコマンドが具体的である。
- 一般論ではなく、その repo やログに即した提案になっている。

# Acceptable Variants
- 結論が1つに定まらない場合でも、最有力仮説と次の確認手順があれば可。

# Common Failure Patterns
- 証拠なしで断定する。
- ローカル制約を見落とす。
- どの repo にも当てはまる抽象論だけで終わる。

# Minimal Pass Line
- 有力な見立てと、それを進める具体的な次の一手が示されていること。

# Notes For Evaluator
- 不確実性の扱いが誠実かを重視する。
