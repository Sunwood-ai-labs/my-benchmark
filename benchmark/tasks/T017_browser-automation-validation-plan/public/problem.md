# Goal
UI 変更を本当に動かしたかを確認するための短いブラウザ検証計画を立てる。

# Background
利用者は『ちゃんと動かして見たのか』を気にしており、ローカル smoke plan が必要である。

# Input Context
- UI ファイル、起動コマンド、主要フローの説明が与えられる。

# User Prompt
短時間で回せるローカル browser validation の手順を作り、何を見れば pass/fail かを明確にしてください。

# Constraints
- 現実的に1セッションで回せる計画にすること。

# Deliverables
- 手順。
- 観測ポイント。
