# Accepted Fix

構造を改善しつつ、利用者から見える挙動を変えないことを重視する。
- 担当範囲だけが改善され、周辺差分がない。

# Minimum Pass

- 利用者から見える振る舞いを保ったまま、少なくとも一段階読みやすくなっていること。

# Acceptable Variants

- 関数抽出や小さな分割など、安全な改善なら形は問わない。

# Common Failures

- 整理のつもりで挙動が変わっている。
- 既存チェックが壊れている。
- リファクタを口実に設計変更へ広げている。

# Evaluator Notes

- きれいさより、壊さなさを優先して見る。
