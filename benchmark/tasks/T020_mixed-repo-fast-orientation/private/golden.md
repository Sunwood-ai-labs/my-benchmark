# Accepted Fix

ローカル証拠に基づいて状況を説明し、次に取るべき行動を明確にできるかを見る。
- 正しい subsystem と妥当な検証経路を短く挙げられる。

# Minimum Pass

- 有力な見立てと、それを進める具体的な次の一手が示されていること。

# Acceptable Variants

- 結論が1つに定まらない場合でも、最有力仮説と次の確認手順があれば可。

# Common Failures

- 証拠なしで断定する。
- ローカル制約を見落とす。
- どの repo にも当てはまる抽象論だけで終わる。

# Evaluator Notes

- 不確実性の扱いが誠実かを重視する。
